Type Indexed Products

  • We can take type driven operation further if we introduce a constraint, such that each type can only occur once in an HList. This is effectively a type-indexed-product.

  • We can use a type class to create the constraint:
    HFreeType e l
    instance HFreeType e HNil
    instance (TypeNotEq e e',HFreeType e l)
       => HFreeType e (HCons e' l)
  • To allow operations on TIPs to be overloaded on existing operators, we create the new type:    newtype TIP l = TIP l { unTIP :: l }

  • We can then lift HCons to TIPs applying the type uniqueness constraint.

  • We can also define type-indexed-coproducts, see paper.
