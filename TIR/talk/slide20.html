Summary

  • There are many types of heterogeneous collection, we provide a definition for strongly typed HLists, and a collection library.

  • We can ensure HLists are well formed, and perform subtypeing using constraints

  • We have defined extesible records in Haskell, without additional language extensions.

  • There is a widely implementable and a general solution to type equality.

  • The extensible records can be used to represent database queries, and support a wide range of opertaions (more examples of which are given in the paper).

  • We have shown how overlapping instances and functional dependancies can be combined to do something useful.

  • Unfortunately we have no time to mention how these techniques can be used to represent XML documents, nor to explain how we can completely capture the semantics of O'Caml in Haskell using the extensible records!
