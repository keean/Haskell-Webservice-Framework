Implementing HTypeEqBool

  • Can be implemented using fundeps such that all platforms (which support multi-parameter type classes and fundeps) can support it.

  • To do this we map each type to a Peano number, then compare Peano numbers. This is similar to the way the 'Typeable' class works, but comparisons are done at the type-level.

  • We cannot use Typeable however as 'typeOf' is evaluated at run-time, not compile time.

  • It is possible to define a generalised version, but this requires GHC's lazy instance selection.

  • We would like to suggest that where HTypeEqBool cannot be defined easily, it should be provided as a primitive, this will also require type-level definitions for True and False.
