Type Driven Operations ...

  • So far we have been dealing with operations that whilst acting on a heterogeneous list, are equivalent to operations on homogeneous lists. However, there are operations not possible on `normal' lists, those that are type driven.

  • We have defined a group of related type driven opperations:

    • hOccursMany - return all the occurances of a type
    • hOccursMany1 - return all (but at least one)
    • hOccursOpt - return Just first occurance or Nothing
    • hOccursFst - return first occurance of one or more
    • hOccurs - establish there is exactly one occurance of type and return value

  • To implement these functions we have to deal with overlapping instances, but as we do not require fundeps, these can be implemented on both GHC and Hugs without problems.
