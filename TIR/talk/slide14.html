Extensible Records ...

  • To allow unique record labels to be associated with freely typed values we pair two HLists together. One HList is constrained to contain only unique types (the keys) the other is an unconstrained HList (the values).

  • For record lookup we use type driven operations on the keys, producing an index which is used with array access to fetch the value.

  • With suitable extra definitions, these extensible records can be made to look very much like normal Haskell records:
    myUnpricedAnimal = Key .=. (42::Integer)
       .*. Name .=. "Angus"
       .*. Breed .=. Cow
       .*. emptyRecord
