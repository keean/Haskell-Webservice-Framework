Database Programming ...

  • Using the extensible records we have just defined, we can represent a database schema, or the result of a query.

  • We have operations to concatenate records, and to project a subset of the elements into a new record.

  • Using these properties we can define a relation to be a record of attribute definitions with Haskell labels, plus some additional information (like source table, and row related restrict information).

  • The type information in the extensible record is effectively a 'wave-front', we maintain type information only for the current relation, as relations get pushed deeper, they can no longer be modified so type information is unnecessary.
